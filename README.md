# 9. プロセス間で共有

## 概要

**インタープリター変数**は19 R5で非推奨になり，プロジェクトモードでは**フィールドの項目リスト**も廃止されました。代わりに**共有シングルトン**を使用することができます。**シングルトン**は，プロセスにつき`1`個しかインスタンス化されないオブジェクトのことで，アプリケーション全体で`1`個しかインスタンス化されないオブジェクトのことを共有シングルトンと呼びます。

どのプロセスからでもアクセスできるという点では`Storage`に似ていますが，シングルトンは目的別にオブジェクトをわけて管理できる点が優れています。さらに，メンバー関数という形でオブジェクト固有のロジックを実装することができます。`me`あるいは`new()`でコンストラクターが呼ばれますが，どのタイミングで初期化しても構わないので，未定義エラーになることがありません。クエリの回数を最少限に抑え，同一のコレクションやオブジェクトを何度も作っては解放するということもなく，パフォーマンスを向上しつつ，メモリの断片化を防止することができます。
